      概述

      • 当前版本：3.1.3
      • 修改时间：2021年7月26日
      • 官方文档：https://docs.spring.io/spring-cloud-stream/docs/3.1.3/reference/html/
      • 源码仓库：https://github.com/spring-cloud/spring-cloud-stream

      Spring Cloud Stream是一个框架，用于构建高度可扩展的事件驱动的微服务，与共享消息系统相连。

      该框架提供了一个灵活的编程模型，该模型建立在已经建立和熟悉的Spring成语和最佳实践之上，包括对持久性pub/sub语义、消费者组和有状态分区的支持。

      1. 序言#

      1.1. Spring的数据整合之旅简史#

      Spring的数据集成之旅始于Spring Integration。通过它的编程模型，它提供了一致的开发者体验，以构建能够拥抱企业集成模式的应用程序，从而与外部系统（如数据库、消息代理等）进行连接。

      快进到云时代，微服务已经在企业环境中变得非常突出。Spring Boot改变了开发者构建应用程序的方式。有了Spring的编程模型和Spring Boot处理的运行时责任，开发独立的、基于Spring的生产级微服务就变得天衣无缝。

      为了将其扩展到数据集成工作负载，Spring Integration和Spring Boot被整合成一个新项目。Spring Cloud Stream由此诞生。

      通过Spring Cloud Stream，开发人员可以。

      • 孤立地构建、测试和部署以数据为中心的应用程序。
      • 应用现代微服务架构模式，包括通过消息传递进行组合。
      • 用以事件为中心的思维解耦应用程序的责任。一个事件可以代表在时间上发生的事情，下游的消费者应用程序可以对其作出反应，而不知道它的来源或生产者的身份。
      • 将业务逻辑移植到消息代理上（如RabbitMQ、Apache Kafka、Amazon Kinesis）。
      • 依靠该框架的自动内容类型支持来实现常见的使用情况。扩展到不同的数据转换类型是可能的。

      1.2. 快速开始#

      按照这个三步指南，你可以在5分钟内尝试Spring Cloud Stream，甚至在你进入任何细节之前。

      我们将向你展示如何创建一个Spring Cloud Stream应用程序，接收来自你所选择的消息中间件的消息（后面会有更多介绍），并将收到的消息记录到控制台。我们称它为LoggingConsumer。虽然不是很实用，但它对一些主要的概念和抽象提供了一个很好的介绍，使其更容易消化本用户指南的其余部分。

      这三个步骤如下。

      1. 通过使用Spring Initializr创建一个示例应用程序
      2. 将项目导入你的IDE
      3. 添加消息处理程序、构建和运行

      1.2.1. 通过使用Spring Initializr创建一个示例应用程序#

      要开始，请访问Spring Initializr。从那里，你可以生成我们的 LoggingConsumer 应用程序。要做到这一点。

      1. 在Dependencies部分，输入入stream。当 Cloud Stream 选项出现时，选择它。

      2. 输入 kafka 或 rabbit。

      3. 选择 Kafka 或 RabbitMQ。 基本上，你选择你的应用程序所绑定的消息传递中间件。我们建议使用你已经安装的或者对安装和运行感到更舒服的那个。另外，正如你从Initilaizer屏幕上看到的，还有一些其他选项可以选择。例如，你可以选择Gradle作为你的构建工具，而不是Maven（默认）。

      4. 在Artifact字段中，输入 logging-consumer。 Artifact字段的值成为应用程序名称。如果你选择RabbitMQ作为中间件，你的Spring Initializr现在应该如下(这里应该有一张图，但是官方的图挂了)。

      5. 单击 Generate 按钮。 这样做会将生成的项目的压缩版本下载到您的硬盘驱动器。

      6. 将文件解压缩到要用作项目目录的文件夹中。

      我们鼓励你去探索Spring Initializr中的许多可能性。它可以让你创建许多不同类型的Spring应用程序。

      1.2.2. 把工程导入到你的IDE#

      现在你可以把项目导入你的IDE。请记住，根据不同的IDE，你可能需要遵循特定的导入程序。例如，根据项目的生成方式（Maven或Gradle），您可能需要遵循特定的导入程序（例如，在Eclipse或STS，您需要使用File → Import → Maven → Existing Maven Project）。

      导入后，该项目必须没有任何形式的错误。另外，src/main/java应包含com.example.loggingconsumer.LoggingConsumerApplication。

      从技术上讲，此时，你可以运行应用程序的主类。它已经是一个有效的Spring Boot应用程序。但是，它并没有做任何事情，所以我们要添加一些代码。

      1.2.3. 添加一个Message Handler，构建和运行#

      修改com.example.loggingconsumer.LoggingConsumerApplication类，看起来如下。

      @SpringBootApplication
public class LoggingConsumerApplication {

	public static void main(String[] args) {
		SpringApplication.run(LoggingConsumerApplication.class, args);
	}

	@Bean
	public Consumer<Person> log() {
	    return person -> {
	        System.out.println("Received: " + person);
	    };
	}

	public static class Person {
		private String name;
		public String getName() {
			return name;
		}
		public void setName(String name) {
			this.name = name;
		}
		public String toString() {
			return this.name;
		}
	}
}

      正如你从前面的列表中看到的那样。

      • 我们使用函数式编程模型（见Spring Cloud Function support）来定义一个单一的消息处理程序作为 Consumer。
      • 我们依靠框架惯例将这种处理程序绑定到绑定器所暴露的输入目标绑定。

      这样做也让你看到框架的一个核心特征。它试图将传入的消息有效payloads自动转换为Person类型。

      现在你有一个功能齐全的Spring Cloud Stream应用程序，它确实在监听消息。从这里开始，为了简单起见，我们假设您在 step one 中选择了 RabbitMQ。假设您已经安装并运行了 RabbitMQ，您可以在 IDE 中运行它的 main 方法来启动该应用程序。

      您应该看到以下输出。

      --- [ main] c.s.b.r.p.RabbitExchangeQueueProvisioner : declaring queue for inbound: input.anonymous.CbMIwdkJSBO1ZoPDOtHtCg, bound to: input
	--- [ main] o.s.a.r.c.CachingConnectionFactory       : Attempting to connect to: [localhost:5672]
	--- [ main] o.s.a.r.c.CachingConnectionFactory       : Created new connection: rabbitConnectionFactory#2a3a299:0/SimpleConnection@66c83fc8. . .
	. . .
	--- [ main] o.s.i.a.i.AmqpInboundChannelAdapter      : started inbound.input.anonymous.CbMIwdkJSBO1ZoPDOtHtCg
	. . .
	--- [ main] c.e.l.LoggingConsumerApplication         : Started LoggingConsumerApplication in 2.531 seconds (JVM running for 2.897)

      转到 RabbitMQ 管理控制台或任何其他 RabbitMQ 客户端并向 input.anonymous.CbMIwdkJSBO1ZoPDOtHtCg 发送一条消息。anonymous.CbMIwdkJSBO1ZoPDOtHtCg部分代表组名，并且是生成的，所以它在你的环境中一定会有所不同。对于更可预测的东西，你可以通过设置spring.cloud.bindings.input.group=hello（或任何你喜欢的名字）来使用一个明确的组名。

      消息的内容应该是Person类的JSON表示，如下所示。

      {"name": "Sam Spade"}。

      然后，在你的控制台，你应该看到。

      Received: Sam Spade

      你也可以将你的应用程序建立并打包成一个启动jar（通过使用./mvnw clean install），并通过使用java -jar命令运行建立的JAR。

      现在你有一个可以工作的（尽管是非常基本的）Spring Cloud Stream应用程序。
